# 강의자료 등록 / 수정

코디움랩 워크스페이스의 강의자료(`materials`)를 등록하거나 수정하는 자동화 커맨드.
LLM과 대화로 작성한 강의자료를 libSQL(Turso) DB 에 직접 저장한다.

- `id` 없음 → **새 자료 등록 (INSERT)**
- `id` 있음 → **기존 자료 수정 (UPDATE)** — 전달된 필드만 갱신

> 앱 안의 [자료 관리] 화면에서도 동일하게 등록·편집할 수 있습니다. 이 커맨드는
> "대화로 자료를 만들고 바로 등록"하는 흐름을 위한 보조 경로입니다.

---

## ⚠️ 자동 실행 규칙

1. **질문 금지**: 입력 JSON 을 그대로 신뢰합니다.
2. **단일 호출**: 아래 bash 를 1회 실행하고 stdout JSON 을 그대로 반환합니다.
3. **JSON만 반환**: `{"id":"...","ok":true,"action":"created"|"updated"}` 형식.

---

## 입력 (stdin JSON 단일 객체)

| 필드 | 필수 | 설명 |
|---|---|---|
| `id` |  | 있으면 수정 모드 |
| `title` | ✅(신규) | 자료 제목 |
| `summary` |  | 한 줄 요약 (목록 카드에 노출) |
| `body` |  | 본문 마크다운 |
| `status` |  | `draft` / `published` / `archived` (기본 `draft`) |
| `access` |  | `public`(로그인 전체) / `restricted`(권한자만, 기본) |
| `category` |  | 분류 |
| `tags` |  | string 배열 |
| `authorName` |  | 작성자명 |

> 접근권한(특정 사용자 × 기간) 부여는 앱의 [자료 관리 → 접근권한] 에서 합니다.
> 신규 자료는 기본 `restricted` + `draft` 이므로, 공개하려면 `status: "published"` 로 두고
> 권한을 부여하거나 `access: "public"` 으로 등록하세요.

---

## 실행 지시

다음을 **정확히 1회** 실행하고 stdout JSON 을 반환하세요. (`<INPUT_JSON>` 자리에 입력을 치환)

```bash
echo '<INPUT_JSON>' | node --env-file-if-exists=.env.local scripts/register-material.mjs
```

한국어가 깨지지 않도록 single-quote 로 감싸세요.

## 응답 (stdout)

```json
{"id": "uuid-...", "ok": true, "action": "created", "title": "..."}
```

실패는 stderr + 비정상 종료.
